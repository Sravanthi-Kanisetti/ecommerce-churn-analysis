Project Flow

PHASE 01
Dataset Generation
Synthetic Dataset — Python (pandas + numpy)
🐍
1,05,000 rows 34 columns Jan 2021 – Jun 2024 Indian ecommerce context 15+ quality issues injected

Realistic data simulating an Indian ecommerce platform — customers, orders, sessions, churn dates, membership tiers, preferred devices, payment methods, complaints, and return behaviour. Intentional data quality issues were injected to mimic real-world messy data: null emails, invalid ages, negative spend, membership typos, duplicate records, mixed gender formats, future signup dates, and impossible return counts.

PHASE 02
Data Exploration
Raw Data Profiling — BigQuery SQL
🔍
NULL counts Invalid value detection Duplicate check Categorical inconsistencies Date anomalies Impossible values

Systematic profiling of every column before any transformation. Issues found:

Null emails ~2,400
Invalid ages ~500
Negative spend values 182
Duplicate records ~1,575
Negative days_since_purchase ~840
Future signup dates 50
Impossible returns > orders ~1,050
Membership typos multiple
PHASE 03
Data Cleaning
Production Table — CREATE OR REPLACE TABLE clean_customers
🧹
CASE WHEN logic REGEXP_REPLACE INITCAP / TRIM NULLIF / COALESCE ROW_NUMBER + QUALIFY ABS() for negatives NTILE window fn

Raw table never modified — all cleaning writes to a new clean_customers table. Deduplication via ROW_NUMBER() OVER PARTITION BY email + QUALIFY row_num = 1. Derived columns added during cleaning: age_group, recency_bucket, is_churned, full_name, standardised membership tier spellings, phone normalisation to last 10 digits.
